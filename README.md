# 🧞 Mr. Meeseeks — AGENTS.md Demo Project

> *"I'm Mr. Meeseeks! Look at me!"*

A hands-on demo showing how **AGENTS.md** and **Skills** standardize code generated by GitHub Copilot across different models, same quality bar.

This project contains a small **helper functions library** built with TypeScript, tested with Vitest, and documented with VitePress. A custom **scoring script** rates generated code on 12 quality criteria so you can see the before/after difference in hard numbers.

---

## Quick Start

```bash
npm install
npm run test        # Run all tests (should pass)
npm run dev         # Start VitePress docs at localhost:5173
npm run score -- src/helpers/<helper>.ts   # Score a helper
```

## The Demo (Step-by-Step)

### What You'll Show

1. **Round 1** — Generate a helper **without** AGENTS.md → Score it
2. **Round 2** — Generate the same helper **with** AGENTS.md → Score it
3. **Round 3** — Generate using the **`/create-helper` skill** → Score it
4. **Compare** the three scores side-by-side

### Preparation

```bash
npm install
npm run test   # Verify everything works
```

### Round 1: Without AGENTS.md

Temporarily disable AGENTS.md:

```bash
mv AGENTS.md AGENTS.md.bak
mv .github/skills .github/skills.bak
```

Open Copilot Chat and use this prompt:

> Create a helper function called `truncateText` that truncates a string to a given max length and adds an ellipsis if truncated. Put it in `sandbox/truncateText.ts` with its test file `sandbox/truncateText.test.ts`.

After Copilot generates the files, score them:

```bash
npm run score -- sandbox/truncateText.ts
```

📸 **Screenshot the score** — this is your "before".

### Round 2: With AGENTS.md

Restore AGENTS.md:

```bash
mv AGENTS.md.bak AGENTS.md
mv .github/skills.bak .github/skills
```

**Start a new Copilot chat session** (important — clear context). Use the exact same prompt:

> Create a helper function called `truncateText` that truncates a string to a given max length and adds an ellipsis if truncated. Put it in `sandbox/truncateTextV2.ts` with its test file `sandbox/truncateTextV2.test.ts`.

Score it:

```bash
npm run score -- sandbox/truncateTextV2.ts
```

Compare side-by-side:

```bash
npm run score -- sandbox/truncateText.ts sandbox/truncateTextV2.ts
```

📸 **Screenshot the comparison** — this is your "with AGENTS.md" improvement.

### Round 3: With the Skill (Bonus)

In Copilot Chat, type `/create-helper` and use this prompt:

> A function that truncates text to N characters with ellipsis

The skill will guide Copilot through the full workflow: function, test, docs, barrel export. Save the output to `sandbox/truncateTextV3.ts`.

```bash
npm run score -- sandbox/truncateTextV3.ts
```

And the final comparison:

```bash
npm run score -- sandbox/truncateText.ts sandbox/truncateTextV3.ts
```

## Scoring Criteria (12 points)

The scoring script checks these criteria — each is pass/fail (1 or 0):

| # | Criterion | What It Checks |
|---|-----------|----------------|
| 1 | JSDoc block present | Has a `/** ... */` comment block |
| 2 | `@description` tag | Documents what the function does |
| 3 | `@param` for each parameter | Every parameter is documented |
| 4 | `@returns` tag | Return value is documented |
| 5 | `@example` (at least 2) | Usage examples in JSDoc |
| 6 | `@throws` tag | Error conditions documented |
| 7 | Input validation (TypeError) | Validates inputs and throws TypeError |
| 8 | No `any` type | No `any` in type annotations |
| 9 | Explicit return type | Function has `: returnType` |
| 10 | Test file exists | Co-located `.test.ts` file |
| 11 | Tests use describe/it | Structured test organization |
| 12 | Tests cover errors (toThrow) | Error scenarios are tested |

## How It Works

### AGENTS.md

The `AGENTS.md` file at the project root is automatically read by GitHub Copilot (in VS Code) whenever it generates code in this workspace. It contains explicit rules for:

- **JSDoc format** — exactly which tags are required and their format
- **Input validation** — how to validate params and format error messages
- **Type safety** — no `any`, explicit return types, `readonly` arrays
- **Testing structure** — `describe`/`it` blocks, minimum test counts
- **File organization** — one function per file, barrel exports, co-located tests

Without AGENTS.md, Copilot uses its general training data, which produces functional but inconsistent code. With AGENTS.md, every model (GPT-4o, Claude, Gemini) converges on the same standard.

### Skills

The `/create-helper` skill goes further — it provides a **step-by-step workflow** that the agent follows:

1. Create the function file (with template)
2. Create the test file (with required structure)
3. Update barrel export
4. Create VitePress documentation
5. Run the quality scorer

This ensures not just the code quality, but the **completeness** of the deliverable.

### The Gold Standard

The helpers in `src/helpers/` are built **with** AGENTS.md guidance and should score 12/12. They serve as the reference implementation the team can study.

## Multi-Model Testing

For maximum demo impact, try the same prompt with different Copilot models:

1. Switch to **GPT-4o** → generate → score
2. Switch to **Claude Sonnet** → generate → score  
3. Switch to **Gemini** → generate → score

**Without AGENTS.md**: scores will vary significantly between models.  
**With AGENTS.md**: scores converge toward 12/12 regardless of model.

This is the key insight: **AGENTS.md normalizes quality across models.**
